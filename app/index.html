Simple Build Tool og Play Framework!

singh@iterate.no

Vi var så heldige å få velge teknologistack under et prosjekt høsten 2013. Vågale som vi var valgte vi teknologier som vi hadde lite kjennskap til, men vi visste at dette var vår sjanse til å lære! Valget falt på Play som webrammeverk med Simple Build Tool (SBT) som byggeverktøy, Angular som frontendrammeverk og Scala som programmeringsspråk.
Foredraget kommer til å dreie seg mest om erfaringer gjort med SBT og hvordan dette spiller med Play. I vårt prosjekt skulle det være funksjonalitet både for administratorbrukere og vanlig brukere. Med SBT lagde vi flere moduler som var uavhengige av hverandre, men som også delte felles kode. Hver modul hadde sin egen ruter hvor inngangen til disse ble styrt av en felles ruter. Kom og lær mer om våre erfaringer rundt dette!

Automated testing with Spock

andrew@iterate.no

During the last 8 months I have been working with the test team at Norwegian to create a tool to help with the writing of Automated tests.
In order to do this I needed to integrate with several Web Portals, SOAP services and JMX services to provide a Fluid API, which could be used by existing members of the test team. This talk will describe the process, explain some of the techniques used and give some examples of how I have made use of the Spock test framework, together with some of the language features of Groovy to provide a test framework which can be used to both document and test core functionality within the Norwegian IT system.

Java 8: Return of the Java

vraalsen@iterate.no

Java 8 is about to unleash Lambdas upon the world! Prepare for what's coming!
Java the platform is arguably stronger than ever, but the language itself is starting to show it's age. The last major new language feature was generics, introduced 10 years ago. This is all about to change. Java 8 is poised to bring us the largest language changes Java has ever seen. We will finally be delivered to the promised land of functional programming! Learn what new powers features such as Lambdas, Extension Methods and Streams give us. We'll also compare briefly with other languages on the JVM such as Scala and Clojure.

Show - don' tell!

haugeto@iterate.no
~ This is a workshop ~

Kan du formidle et budskap så sterkt at mottageren til slutt tenker som deg?
For utvikleren som vil få kunden til å kode bedre. For Scrum-masteren som vil ha produkteieren til eksperimentere mer. For selgeren som vil få avdelingslederen vekk fra gammel innkjøpsmodell. Da sår du frø som spirer hos mottageren helt til det er han eller hun som bryter ut "vi må jobbe mer iterativt!" I denne workshoppen trener vi på noen enkle kommunikasjonsutfordringer, som gjør at vi sammen kan finne frem til gode eksempler på påvirkning av kunder, både i oppdrag, markedsføring og salg. Kort sagt: En workshop for alle som ønsker å påvirke.

palru@iterate.no

Ytelse er viktig på web, men treghet introduseres i hvert ledd fra der dataen finnes og fraktes opp til brukeren.

Hvordan oppstår etablerte sannheter?

kim@iterate.no

Etablerte sannheter er sannheter som ikke lenger er gjeldende, men som fortsatt lever i form av at de er hensyntatt. Dette kan påvirke alt fra kildekode til organisasjonsform og produktprotefølje i en virksomhet.
Det er veldig viktig å ha et bevisst forhold til dette for å kunne utfordre og fjerne etablerte sannheter. Det er mange årsaker til at sannheten dukket opp i første omgang, men årsaken til at de blir stående, selv når de potensielt koster skjorta å forholde seg til.. Dette foredraget tar for seg hvordan slike sannheter blir etablert og vedvarer.
